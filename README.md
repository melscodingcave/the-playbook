# 📋 the-playbook

A no-code repository that documents how I think, plan, and work — not just what I build.

Most engineering portfolios are collections of code. This one includes the reasoning behind the code: the decisions made, the tradeoffs considered, the scope calls, and the process that connects requirements to working software.

---

## 🎯 What This Repo Is

`the-playbook` exists to showcase the parts of software engineering that don't show up in a diff:

- How I approach domain modeling before writing a single line of code
- How I use AI as a collaborator without losing ownership of my decisions
- How I think about scope, tradeoffs, and intentional constraints
- How I document process the same way I document code — as a first-class artifact
- How lessons learned in one project carry forward into the next
